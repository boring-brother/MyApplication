**问题描述**: ClientActivity 中使用了 API 30+ 的沉浸式 API，但项目 minSdk 为 24

**原代码**:
```kotlin
window.insetsController?.apply {
    hide(android.view.WindowInsets.Type.statusBars() or android.view.WindowInsets.Type.navigationBars())
    systemBarsBehavior = WindowInsetsController.BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE
}
```

**修复方案**: 使用兼容旧版本的沉浸式方案
```kotlin
@Suppress("DEPRECATION")
private fun enableFullScreen() {
    window.decorView.systemUiVisibility = (
        View.SYSTEM_UI_FLAG_IMMERSIVE_STICKY
        or View.SYSTEM_UI_FLAG_LAYOUT_STABLE
        or View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION
        or View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN
        or View.SYSTEM_UI_FLAG_HIDE_NAVIGATION
        or View.SYSTEM_UI_FLAG_FULLSCREEN
    )
}
```

**状态**: ✅ 已修复

### 2. 辅助功能权限警告 (已处理)

**问题描述**: BIND_ACCESSIBILITY_SERVICE 权限标记为系统权限

**修复方案**: 添加 tools:ignore 忽略警告（该权限为辅助功能服务必需）
```xml
<uses-permission android:name="android.permission.BIND_ACCESSIBILITY_SERVICE"
    tools:ignore="ProtectedPermissions" />
```

**状态**: ✅ 已处理（该权限为必要权限）

---

### 3. 冗余 label 属性 (已修复)

**问题描述**: MainActivity 中重复声明了 label 属性

**修复方案**: 移除冗余的 label 声明
```xml
<activity
    android:name=".MainActivity"
    android:exported="true"
    android:theme="@style/Theme.MyApplication">
```

**状态**: ✅ 已修复

### 4. 屏幕方向锁定 (已优化)

**问题描述**: ClientActivity 锁定了竖屏方向，在 Android 16+ 可能被忽略

**修复方案**: 移除屏幕方向锁定，让应用自适应
```xml
<activity
    android:name=".ui.ClientActivity"
    android:exported="false"
    android:theme="@style/Theme.MyApplication" />
```

**状态**: ✅ 已优化

---

### 5. 触摸事件可访问性警告 (已处理)

**问题描述**: setOnTouchListener 没有调用 performClick

**修复方案**: 添加 @Suppress 注解（该场景下不需要 performClick）
```kotlin
@Suppress("ClickableViewAccessibility")
private fun setupTouchOverlay() {
    // 触摸事件处理逻辑
}
```

**状态**: ✅ 已处理（该场景为连续触摸操作，不需要 performClick）
